# 关卡编辑器说明

本文档整理当前项目中关卡编辑器和交互/可破坏物系统的实际功能、使用方式、保存机制和已知限制。

相关源码：

- `src/main.js`
- `src/ui/LevelEditorUI.js`
- `src/scene/InteractiveObjectManager.js`

## 1. 入口

1. 进入游戏。
2. 打开 ESC 菜单。
3. 点击关卡编辑器入口。

当前编辑器会编辑“当前正在游玩的关卡”。例如当前关卡是 `city_circuit`，保存数据就会写入 `city_circuit` 对应的编辑布局；当前关卡是 `city_circuit_01`，则写入 `city_circuit_01`。

## 2. 编辑模式行为

进入关卡编辑器后：

- 游戏会进入上帝视角编辑模式。
- 玩家车辆会被冻结，不会继续行驶。
- 交通车、警车、交互物碰撞在编辑模式下会暂停处理。
- 360 观察按钮会隐藏。
- 退出编辑器后，相机会恢复到车辆跟随视角。
- 退出后会恢复正常驾驶输入。

## 3. 上帝视角操作

编辑模式下，移动按键控制的是编辑相机，不再控制车辆。

| 操作 | 作用 |
| --- | --- |
| `W` / `↑` | 编辑相机向前移动 |
| `S` / `↓` | 编辑相机向后移动 |
| `A` / `←` | 编辑相机向左移动 |
| `D` / `→` | 编辑相机向右移动 |
| `Shift` / 氮气按钮 | 加快相机移动 |
| `Space` / 漂移按钮 | 降低相机移动速度 |

补充说明：

- 如果正在编辑输入框、下拉框或文本框，键盘移动会被忽略，避免改数值时相机移动。
- 编辑相机会尝试贴近当前道路高度，保持在地面上方约 `1.5` 米。
- 按重置键时，编辑相机目标会回到车辆附近。

## 4. 编辑器界面

编辑器界面分为顶部工具条和底部编辑面板。

顶部工具条包含：

- 当前关卡 ID。
- 当前交互物数量。
- 状态提示文字。
- `Save` 按钮。
- `Save & Exit` 按钮。

底部编辑面板包含：

- 左侧物体库。
- 右侧属性面板。

注意：当前 `LevelEditorUI.js` 中部分中文提示文本存在编码乱码，但功能仍可使用。

## 5. 可放置物体类型

当前可放置物体由 `INTERACTIVE_OBJECT_TYPES` 定义。

| 类型 ID | 名称 | 质量 | 坚固度 | 碰撞半径 | 可破坏 | 特殊效果 |
| --- | --- | ---: | ---: | ---: | --- | --- |
| `traffic_cone` | Traffic Cone | 8 | 18 | 0.85 | 是 | 无 |
| `barrel` | Impact Barrel | 28 | 42 | 1.05 | 是 | 无 |
| `wood_crate` | Wood Crate | 36 | 58 | 1.15 | 是 | 无 |
| `tire_stack` | Tire Stack | 48 | 75 | 1.20 | 否 | 无 |
| `road_barrier` | Road Barrier | 120 | 160 | 1.85 | 否 | 无 |
| `breakable_sign` | Breakable Sign | 18 | 32 | 0.90 | 是 | 无 |
| `boost_pad` | Boost Pad | 0 | 999 | 1.70 | 否 | 加速 |

## 6. 放置物体

1. 在底部物体库中选择物体类型。
2. 鼠标或触摸移动到场景地面。
3. 场景中会出现半透明预览。
4. 点击地面即可放置物体。
5. 新放置的物体会自动被选中。

放置规则：

- 默认开启 `snapToGround`。
- 开启吸附地面时，会尝试把物体吸附到道路/地面高度。
- 如果无法取得道路高度，会保留原位置或使用默认高度。

## 7. 选择和移动物体

- 点击场景中已有交互物可以选中。
- 选中后按住拖动，可以移动物体。
- 拖动时会更新物体位置。
- 拖动时默认保持地面吸附。
- 选中物体后，右侧属性面板会显示该物体参数。

## 8. 属性面板

选中物体后，可编辑以下属性：

| 属性 | 说明 | 范围/类型 |
| --- | --- | --- |
| `type` | 物体类型 | 物体库中的类型 ID |
| `mass` | 质量，影响撞击反馈和伤害计算 | `0` 到 `500` |
| `durability` | 坚固度，可破坏物的生命值 | `1` 到 `999` |
| `collisionRadius` | 简化碰撞半径 | `0.25` 到 `5` |
| `scale` | 模型缩放 | `0.35` 到 `4` |
| `rotationY` | Y 轴朝向 | `-6.28` 到 `6.28` |
| `snapToGround` | 是否吸附地面 | 开/关 |
| `breakable` | 是否可破坏 | 开/关 |
| `respawn` | 破坏后是否重生 | 开/关 |
| `respawnSeconds` | 重生时间 | `1` 到 `120` 秒 |

属性修改后会标记为未保存，需要点击 `Save` 或 `Save & Exit` 保存。

## 9. 复制和删除

右侧属性面板提供：

- `Duplicate`：复制当前选中物体。
- `Delete`：删除当前选中物体。

复制规则：

- 复制体会基于当前物体参数生成。
- 复制体位置会在原物体基础上偏移约 `2` 米，避免完全重叠。

## 10. 保存机制

顶部按钮：

- `Save`：保存当前关卡编辑数据。
- `Save & Exit`：保存并退出编辑器。

自动保存：

- 隐藏/退出编辑器时，如果存在未保存改动，会自动尝试保存。

保存反馈：

- 保存成功时 HUD 会提示 `SAVED`。
- 保存失败时 HUD 会提示 `SAVE FAILED`。

## 11. 保存位置和数据结构

当前编辑器不是写入项目内的 JSON 文件，而是写入浏览器/游戏本地存储。

存储方式：

- `localStorage`
- Key：`cargame_level_editor_v1`

数据结构示例：

```json
{
  "tracks": {
    "city_circuit": [
      {
        "id": "traffic_cone-abc-1",
        "type": "traffic_cone",
        "position": { "x": 0, "y": 0, "z": 0 },
        "rotationY": 0,
        "scale": 1,
        "snapToGround": true,
        "mass": 8,
        "durability": 18,
        "collisionRadius": 0.85,
        "breakable": true,
        "respawn": false,
        "respawnSeconds": 12,
        "effect": "none"
      }
    ],
    "city_circuit_01": []
  }
}
```

重要影响：

- 该数据保存在当前设备和当前 WebView/浏览器环境中。
- 清空应用数据、清空浏览器数据、重新安装 APK 或换设备后，数据可能丢失。
- 当前不会自动进入安装包内的 `static/config` 配置。

## 12. 加载机制

关卡加载时，`InteractiveObjectManager.loadTrack(trackId)` 会执行：

1. 标准化关卡 ID。
2. 清除当前交互物。
3. 从 `localStorage` 读取当前关卡保存的物体布局。
4. 如果存在保存数据，则恢复物体。
5. 如果没有保存数据，则使用默认空布局。

当前默认布局为空，也就是没有内置交互物。

## 13. 游戏内碰撞和破坏逻辑

编辑器保存后的交互物会参与正常游戏碰撞。

碰撞判断：

- 使用简化半径碰撞。
- 计算玩家车辆与物体的 XZ 平面距离。
- 判定半径约为 `collisionRadius * scale + 1.15`。

撞击计算：

- 根据车速和车辆速度计算冲击强度。
- 根据物体质量计算伤害。
- 质量越大，受到的伤害越小。
- 不可破坏物会更强地把车推开。

可破坏物：

- `breakable = true` 且 `durability <= 0` 时会被破坏。
- 破坏后物体隐藏。
- 如果 `respawn = true`，会在 `respawnSeconds` 秒后恢复。

不可破坏物：

- 不会被摧毁。
- 会对车辆产生更明显的反弹和阻挡。

加速板：

- `boost_pad` 的 `effect` 是 `boost`。
- 接触后会给车辆增加速度推力。
- 不走普通破坏逻辑。

粒子效果：

- 撞击、破坏和加速板会生成简单粒子。
- 粒子由 `EffectManager.emitParticles` 生成。

## 14. 当前已知限制

- 编辑器 UI 里部分中文提示文本乱码。
- 保存只在 `localStorage`，不是项目文件级保存。
- 没有关卡选择面板，只能编辑当前正在游玩的关卡。
- 物体模型是代码生成的简易几何体，不是外部 GLB 模型。
- 没有三轴移动/旋转/缩放 Gizmo。
- 没有撤销/重做。
- 属性面板没有直接编辑位置坐标。
- 碰撞是简化半径碰撞，不是精确网格碰撞。
- 物体库无法从配置文件扩展，当前写死在代码里。
- 默认布局为空，没有内置摆放数据。
- 保存数据无法直接随安装包发布。
