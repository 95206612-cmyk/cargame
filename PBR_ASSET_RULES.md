# Street Racer 新 PBR 渲染引擎资产制作规则

本文档用于约束新渲染引擎下的模型、材质、贴图和 GLB 导出。目标是让赛道贴图稳定显示、PBR 材质正确受光、实时阴影正常工作，并兼顾 PC 与 APK 性能。

## 1. 基础坐标与导出规则

- 引擎坐标：Three.js / glTF 标准，`Y` 轴向上，`X/Z` 为地面平面。
- 单位：`1 unit = 1 meter`。建模软件导出前必须应用缩放、旋转和位移。
- 原点：赛道建议放在世界中心附近，整体范围尽量控制在 `±500m` 内，避免过大坐标导致阴影和深度精度下降。
- 文件格式：优先使用 `.glb`，贴图可内嵌，便于打包和 APK 同步。
- 禁止：负缩放、未应用的非等比缩放、极远离原点的模型、隐藏但仍导出的高面数物体。

## 2. 赛道文件拆分

赛道资产按两个文件交付：

| 文件 | 用途 | 是否参与物理/导航 |
| --- | --- | --- |
| `*-track.glb` | 路面、路肩、马路牙子、护栏、墙体、终点线相关可驾驶区域 | 是 |
| `*-scenery.glb` | 建筑、树、灯杆、广告牌、观众席、远景、装饰物 | 是，生成简化场景碰撞 |

示例：

- `city-circuit-track.glb`
- `city-circuit-scenery.glb`

## 3. 命名规则

新引擎会按 Mesh 或 Material 名称自动判断材质类型。建议使用英文小写关键词。

| 类型 | 推荐关键词 |
| --- | --- |
| 沥青/跑道 | `road`, `asphalt`, `track`, `street`, `lane` |
| 马路牙子 | `curb`, `kerb`, `road_trim` |
| 草地/沙地/泥地 | `grass`, `terrain`, `sand`, `dirt`, `ground`, `soil` |
| 护栏/墙体 | `barrier`, `fence`, `wall`, `guardrail`, `railing` |
| 建筑/看台 | `building`, `stadium`, `grandstand`, `pit`, `garage` |
| 金属件 | `metal`, `rail`, `post`, `sign`, `rim`, `hub` |
| 玻璃 | `glass`, `window`, `windscreen`, `windshield` |
| 灯光/发光 | `light`, `lamp`, `neon`, `emissive`, `headlight`, `taillight` |
| 车辆 | `car`, `body`, `paint`, `chassis`, `wheel`, `tire`, `tyre` |

如果命名不规范，引擎会按 `prop` 默认材质处理，可能导致粗糙度、反射和阴影效果不符合预期。

## 4. 面数预算

下面是建议预算，按三角面计算。超过预算不一定不能运行，但会增加加载时间、APK 体积、GPU 压力和阴影开销。

### 整体场景

| 平台 | 推荐总三角面 | 上限建议 |
| --- | ---: | ---: |
| PC 高画质 | `300k - 600k` | `800k` |
| PC 中画质 | `180k - 350k` | `500k` |
| Android / APK | `100k - 220k` | `300k` |

### 赛道与布景

| 资产 | 推荐三角面 | 上限建议 |
| --- | ---: | ---: |
| `*-track.glb` 总体 | `40k - 120k` | `180k` |
| 可驾驶路面 | `15k - 60k` | `90k` |
| 物理/导航路面轮廓 | `5k - 25k` | `40k` |
| 护栏/墙体 | `10k - 50k` | `80k` |
| `*-scenery.glb` 总体 | `120k - 350k` | `500k` |
| 单个近景建筑 | `2k - 20k` | `35k` |
| 单个远景建筑 | `200 - 3k` | `8k` |
| 树/灯杆/广告牌 | `300 - 5k` | `10k` |

### 车辆

| 资产 | 推荐三角面 | 上限建议 |
| --- | ---: | ---: |
| 玩家车辆完整模型 | `25k - 60k` | `90k` |
| 车身 | `15k - 35k` | `55k` |
| 每个轮胎/轮毂 | `2k - 6k` | `10k` |
| AI 车辆 | `3k - 12k` | `20k` |

## 5. 赛道物理与导航规则

- 路面必须是独立 Mesh，不能和草地、墙体、建筑合并成一个 Mesh。
- 路面应为连续闭合赛道，内外边界清晰，避免断裂、重叠、翻面和极小碎三角。
- 推荐赛道宽度：`8m - 18m`。当前导航提取会把半宽限制在约 `3.8m - 10.5m`。
- 路面高度变化要平滑，坡度、桥面、弯道外倾不要出现突然台阶。
- 护栏、墙体、赛道边界必须单独成 Mesh，方便生成碰撞体。
- 场景布景碰撞会被简化成盒体/网格碰撞，不要依赖细碎雕刻面产生精确碰撞。
- 视觉细节可以高，物理轮廓必须简单、连续、少碎面。

## 6. PBR 材质规则

优先使用 glTF 标准 PBR 材质：`MeshStandardMaterial` 或 `MeshPhysicalMaterial`。

| 材质类型 | Metalness | Roughness | 说明 |
| --- | ---: | ---: | --- |
| 干燥沥青 | `0.0 - 0.05` | `0.68 - 0.88` | 不要做成镜面，法线强度要轻 |
| 湿滑沥青 | `0.0 - 0.05` | `0.35 - 0.55` | 可更亮，但不要过曝 |
| 草地/土地 | `0.0` | `0.85 - 1.0` | 低反射，高粗糙 |
| 沙地/泥地 | `0.0` | `0.75 - 0.95` | 颜色不要烘焙强阴影 |
| 护栏金属 | `0.6 - 0.95` | `0.22 - 0.48` | 金属贴图优先控制局部 |
| 车漆 | `0.25 - 0.65` | `0.18 - 0.38` | 推荐 Physical 材质加 clearcoat |
| 玻璃 | `0.0 - 0.05` | `0.02 - 0.12` | 透明材质尽量少用双面 |
| 发光灯牌 | `0.0` | `0.25 - 0.55` | 使用 emissive，不要只靠 baseColor 变亮 |

禁止把灯光阴影、AO、强高光直接烘在 baseColor 里。baseColor 只保留材质本身颜色。

## 7. 贴图通道规则

| 贴图 | 用途 | 色彩空间 |
| --- | --- | --- |
| BaseColor / Albedo | 基础颜色 | sRGB |
| Emissive | 发光颜色 | sRGB |
| Normal | 法线 | Linear / Non-Color |
| Roughness | 粗糙度 | Linear / Non-Color |
| Metalness | 金属度 | Linear / Non-Color |
| AO | 环境遮蔽 | Linear / Non-Color |
| Alpha | 透明度 | Linear / Non-Color |

glTF 标准的 MetallicRoughness 贴图可以使用：

- `G` 通道：Roughness
- `B` 通道：Metalness
- AO 单独使用红色通道

## 8. 贴图尺寸规则

贴图尽量使用 2 的幂尺寸：`512`, `1024`, `2048`, `4096`。

| 资产 | PC 推荐 | Android 推荐 |
| --- | ---: | ---: |
| 整体赛道 atlas | `2048 - 4096` | `2048` |
| 路面平铺贴图 | `2048` | `1024 - 2048` |
| 车辆车身 | `2048 - 4096` | `1024 - 2048` |
| 车辆轮胎/内饰 | `1024 - 2048` | `512 - 1024` |
| 近景建筑 | `1024 - 2048` | `1024` |
| 远景建筑/小道具 | `512 - 1024` | `512` |
| 草地/沙地/泥地平铺 | `1024 - 2048` | `1024` |
| Logo/广告牌 | `1024 - 2048` | `1024` |

除非是非常重要的主视觉资产，不建议使用 `8192` 贴图。APK 版本建议单张贴图不超过 `2048`。

## 9. 贴图格式规则

当前项目最稳妥格式：

| 格式 | 使用场景 |
| --- | --- |
| `.png` | Normal、Roughness、Metalness、AO、Alpha、Logo、文字、红白路肩 |
| `.jpg` / `.jpeg` | BaseColor、草地、沙地、沥青、建筑大面积颜色 |

暂不建议直接交付：

- `.webp`：部分 glTF 流程兼容性不稳定。
- `.ktx2` / Basis：项目当前没有配置 KTX2Loader，后续可升级。
- `.tga`, `.tif`, `.psd`：不适合直接进游戏包。

红白路肩、广告牌文字、品牌 Logo 必须用 PNG 或高质量 JPEG，避免压缩块导致边缘糊掉。

## 10. UV 与 Atlas 规则

- 每个可见 Mesh 必须有 `UV0`。
- AO 如需单独烘焙，建议使用不重叠 `UV2`。
- 平铺材质可以重复 UV；Logo、路肩、广告牌不建议过度拉伸。
- Atlas 图集每个 UV 岛之间至少留 `8px - 16px` padding。
- 路肩红白条不要贴到 atlas 边缘，避免 mipmap 或线性采样串色。
- 如果使用单张大 atlas 并且 UV 岛很密，建议在导出 sampler 中关闭 mipmap，或增加足够 padding。

## 11. 实时光影规则

- 大体积建筑、护栏、车辆、灯杆应允许 cast shadow。
- 路面、草地、地面应 receive shadow。
- 玻璃、透明广告布、粒子、UI 面片不建议 cast shadow。
- 避免大量小物体都投实时阴影。小于 `0.3m` 的细节可不投影。
- 发光材质需要 emissive，但 emissive 本身不会照亮场景；需要真实照明时应搭配 PointLight 或 SpotLight。

## 12. Draw Call 与材质数量

| 资产 | 推荐材质数量 |
| --- | ---: |
| `*-track.glb` | `5 - 20` |
| `*-scenery.glb` | `20 - 80` |
| 玩家车辆 | `6 - 18` |
| AI 车辆 | `3 - 8` |
| 单个小道具 | `1 - 3` |

能共用材质的物体尽量共用。不要给每个护栏柱、每棵树、每个路灯都生成独立材质。

## 13. 文件大小目标

| 文件 | PC 推荐 | Android 推荐 |
| --- | ---: | ---: |
| `*-track.glb` | `< 25 MB` | `< 15 MB` |
| `*-scenery.glb` | `< 80 MB` | `< 40 MB` |
| 玩家车辆 GLB | `< 20 MB` | `< 10 MB` |
| AI 车辆 GLB | `< 5 MB` | `< 3 MB` |

超过目标时，优先处理贴图尺寸、重复材质、不可见背面、远景高面数模型。

## 14. 导出前检查清单

- 模型为 Y-up，单位为米，缩放已应用。
- 路面、护栏、地形、建筑是独立 Mesh。
- Mesh 和 Material 名称包含正确关键词。
- 使用 PBR 材质，不使用旧式 Phong/Lambert 作为最终材质。
- BaseColor/Emissive 为 sRGB，Normal/Roughness/Metalness/AO 为 Linear。
- 贴图尺寸为 2 的幂，APK 不超过 2048。
- UV 没有严重拉伸，atlas 有足够 padding。
- 法线方向正确，无大量重叠面、翻面、隐藏高模。
- 物理相关路面连续、平滑、少碎面。
- GLB 能在 glTF Viewer 中正常显示红白路肩、Logo、草地和建筑贴图。
